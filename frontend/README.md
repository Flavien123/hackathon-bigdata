# Транспортный надзор

Мобильное веб-приложение для подачи жалоб на ситуации в общественном транспорте.

## Настройка для работы с локальным API

1. Создайте файл `.env.local` в корне проекта:
\`\`\`bash
cp .env.local.example .env.local
\`\`\`

2. Откройте `.env.local` и укажите адрес вашего локального API:
\`\`\`env
NEXT_PUBLIC_API_URL=http://127.0.0.1:8000
\`\`\`

Замените порт, если ваш backend API работает на другом порту.

3. Установите зависимости:
\`\`\`bash
npm install
\`\`\`

4. Запустите приложение:
\`\`\`bash
npm run dev
\`\`\`

Приложение будет доступно по адресу `http://localhost:3000`

## API Endpoint

Приложение отправляет POST запросы на `/api/complaints` со следующей структурой:

\`\`\`json
{
  "complaint_text": "string",
  "route_number_qr": "string",
  "bus_number_qr": 0,
  "latitude": 0,
  "longitude": 0
}
\`\`\`

## Возможности

- Ввод текста жалобы
- Сканирование QR-кода автобуса
- Автоматическое определение геолокации
- Загрузка фото и видео
- Нативное поведение на мобильных устройствах
- PWA поддержка
